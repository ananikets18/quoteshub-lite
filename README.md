# Quoteshub Lite - Backend

A lightweight backend API for discovering and sharing quotes, built with AdonisJS.

## Architecture

This is the **backend repository** for Quoteshub. The frontend is maintained separately:

- **Backend (this repo)**: [quoteshub-lite](https://github.com/ananikets18/quoteshub-lite)
- **Frontend**: [quotehub-lite-frontend](https://github.com/ananikets18/quotehub-lite-frontend)

## Structure

```
quoteshub-lite/
├── backend/           # AdonisJS backend application
├── render.yaml        # Render deployment configuration
└── README.md
```

## Tech Stack

- **Framework**: AdonisJS 7
- **Database**: PostgreSQL
- **ORM**: Lucid
- **Authentication**: Token-based auth with sessions
- **Rate Limiting**: Upstash Redis
- **Validation**: VineJS

## Features

- User authentication (signup, login, password reset)
- Quote CRUD operations
- Like & save functionality
- Categories & tags
- User profiles & analytics
- Notifications system
- Admin dashboard
- Rate limiting on all endpoints

## Getting Started

### Prerequisites

- Node.js 20+
- PostgreSQL database
- Redis (optional, for rate limiting)

### Installation

1. Navigate to the backend directory:
   ```bash
   cd backend
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Copy the environment file:
   ```bash
   cp .env.example .env
   ```

4. Generate an app key:
   ```bash
   node ace generate:key
   ```

5. Configure your `.env` file with:
   - Database credentials
   - Redis credentials (optional)
   - Session settings

6. Run migrations:
   ```bash
   node ace migration:run
   ```

### Development

Start the development server:
```bash
npm run dev
```

The API will be available at `http://localhost:3333`

### Testing

Run the test suite:
```bash
npm test
```

### Production Build

Build for production:
```bash
node ace build
```

Start the production server:
```bash
cd build
npm ci --omit=dev
node bin/server.js
```

## API Documentation

The API is RESTful and follows these conventions:

- Base URL: `/api/v1`
- Authentication: Session-based (cookies)
- Rate limiting: Applied to all routes

### Main Endpoints

- `GET /quotes` - List quotes
- `POST /quotes` - Create quote (auth required)
- `GET /quotes/:id` - Get quote details
- `PUT /quotes/:id/like` - Toggle like (auth required)
- `GET /categories` - List categories
- `GET /tags` - List tags
- `POST /auth/signup` - Register
- `POST /auth/login` - Login
- `GET /account/profile` - Get user profile (auth required)

## Deployment

This application is configured for deployment on [Render](https://render.com) using the `render.yaml` configuration file.

### Render Setup

1. Connect this repository to Render
2. The service will auto-deploy using `render.yaml`
3. Configure environment variables in Render dashboard:
   - `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_DATABASE`
   - `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`
   - `CORS_ORIGIN` (your frontend URL)

### Environment Variables

Required for production:

- `NODE_ENV=production`
- `PORT=3333`
- `HOST=0.0.0.0`
- `APP_KEY` (auto-generated by Render)
- `DB_*` (PostgreSQL credentials)
- `SESSION_DRIVER=cookie`
- `CORS_ORIGIN` (frontend URL)
